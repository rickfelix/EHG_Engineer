## Proposed Monitoring Queries

### 1. Validation Score

**SQL** (proposed):
```sql
-- Calculate aggregate validation score
SELECT
  venture_id,
  (problem_validation_pct + solution_validation_pct + willingness_to_pay_pct) / 3 AS overall_validation_score,
  CASE
    WHEN (problem_validation_pct + solution_validation_pct + willingness_to_pay_pct) / 3 >= 75 THEN 'PROCEED'
    WHEN (problem_validation_pct + solution_validation_pct + willingness_to_pay_pct) / 3 < 50 THEN 'KILL'
    ELSE 'REVISE'
  END AS decision
FROM validation_scores
WHERE stage_id = 3
GROUP BY venture_id;
```

**Thresholds**:
- **PROCEED**: ≥75%
- **REVISE**: 50-74%
- **KILL**: <50%

**Alert**: If validation_score < 50%, trigger KILL decision gate

---

### 2. User Interest Level

**SQL** (proposed):
```sql
-- Measure % of users with strong interest
SELECT
  venture_id,
  COUNT(CASE WHEN interest_level = 'strong' THEN 1 END) * 100.0 /
    COUNT(*) AS user_interest_pct
FROM user_interviews
WHERE stage_id = 3
GROUP BY venture_id;
```

**Threshold**: ≥60%
**Alert**: If user_interest < 60%, flag for review in decision gate

---

### 3. Technical Feasibility Score

**SQL** (proposed):
```sql
-- Calculate technical feasibility confidence
SELECT
  venture_id,
  AVG(feasibility_rating) AS technical_feasibility_score
FROM technical_assessments
WHERE stage_id = 3
GROUP BY venture_id;
```

**Threshold**: ≥70%
**Alert**: If feasibility_score < 70%, trigger technical review or REVISE decision

## Recursion Metrics

**Additional metrics for recursion tracking**:

| Metric | Definition | Threshold | Purpose |
|--------|------------|-----------|---------|
| **Recursion count** | Number of times Stage 3 re-executed for venture | ≤3 | Loop prevention |
| **Validation delta** | Change in validation score after recursion | Track trend | Improvement indicator |
| **Recursion resolution time** | Time to complete recursion cycle | ≤2 days | Performance monitoring |

**SQL** (proposed):
```sql
-- Track recursion events and deltas
SELECT
  venture_id,
  recursion_count_for_stage,
  EXTRACT(EPOCH FROM (resolution_time - trigger_time)) / 86400 AS resolution_days,
  (trigger_data->>'original_validation_score')::FLOAT -
    (trigger_data->>'new_validation_score')::FLOAT AS validation_delta
FROM recursion_events
WHERE to_stage = 3;
```
