Protocol

Three steps from full table to fast suite

step 1 / 3

Connect

Point DataDistill at a read-only Postgres source. Nothing is written back.

step 2 / 3

Distill

Choose a target size — the recommended preset is already selected — and DataDistill builds a statistically representative subset.

step 3 / 3

Test

Wire the subset into CI from the dashboard or API and run the same suite, faster.

connected + distilled → your next pipeline run is already step three.

Why not just subset by hand? Manual subsetting is tedious, error-prone, and quietly biases what your tests see. DataDistill preserves means, variance, and distributions mathematically — the subset stays a faithful stand-in for the source.
