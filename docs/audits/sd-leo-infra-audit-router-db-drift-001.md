# Audit: CLAUDE.md Router Twins vs `leo_protocol_sections` DB Drift

**SD:** SD-LEO-INFRA-AUDIT-CONTENT-ROUTER-001
**Date:** 2026-05-29
**Active protocol:** `leo-v4-3-3-ui-parity` (v4.4.1)
**Deliverable type:** Read-only audit. **Fixes are OUT OF SCOPE** — filed as follow-up QF(s); chairman triages.

## What this audits

`CLAUDE.md` (the router) is generated by `scripts/modules/claude-md-generator/file-generators.js::generateClaudeMd()`. Some sections are **hardcoded as inline template literals** in that function (the "twin"); others are **injected from the DB** (`leo_protocol_sections`). Hardcoded twins do NOT auto-update when the DB row changes, so they can drift. This audit pairs each `_removed_sections_note` section (from `scripts/section-file-mapping.json`) with its router twin and its DB row, classifying drift:

- **(a) faithful** — twin accurately condenses the DB canonical.
- **(b) summarizes but missing important content** — condensation drops material rules.
- **(c) contradicts DB** — twin states something the DB row conflicts with.
- **(d) DB row missing entirely** — the referenced section key has no `leo_protocol_sections` row.
- **(e) twin missing entirely** — section removed from the router with no inline summary (acceptable when it lives in CORE/phase files).

## Method

Read `_removed_sections_note` from `section-file-mapping.json` (router block + LEAD/PLAN/EXEC blocks); located hardcoded twins in `file-generators.js`; fetched DB rows via the active protocol (`leo-v4-3-3-ui-parity`) the generator queries; compared content. **Important distinction:** the router's "AUTO-PROCEED Mode" and "Session Initialization" blocks are **DB-injected** (`auto_proceed_router` id 567 at file-generators.js:112; `session_init` id 299 at :157), NOT hardcoded twins — so they cannot drift and are marked N/A.

## Limitations

- "Faithful vs missing-content" is a judgment; classifications here are conservative — (b) is reserved for material omissions (a rule an agent would act on), not mere brevity.
- The LEAD/PLAN/EXEC duplicate-of-CORE sections were classified by the `_removed_sections_note` rationale, not a line-by-line diff of each phase file twin; a deeper pass could confirm each phase generator emits no stale inline copy. The router (highest-traffic file) was audited line-by-line.
